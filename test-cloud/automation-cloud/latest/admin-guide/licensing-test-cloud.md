This page provides information about Test Cloud licensing.

## Licensing plan framework

If you purchased an **Application Testing** licensing plan, you can use Test Cloud.

:::note
Our commercial offering provides a flexible variety of licensing SKUs for how to deploy and use UiPath software. Refer to the [UiPath Licensing page](https://licensing.uipath.com/) for the current list of available SKUs. For Test Cloud, the applicable SKUs are on the **Unified Pricing Plan** tab.
:::

### Free

The Free plan provides the following benefits:

* You can use basic development and attended automation capabilities
* You can use the Free plan for commercial purposes, but direct sign-up is not available. The Free plan becomes accessible only when your current trial or paid licensing plan expires and your account is downgraded.
  :::note
  When your trial or paid licensing plan expires, you are automatically downgraded to the Free licensing plan. This transition causes you to lose access to the special benefits unique to the trial or paid licensing plans. For a comprehensive understanding, refer to the following resources:
  * [License expiration and
  downgrade](https://docs.uipath.com/automation-cloud/automation-cloud/latest/admin-guide/upgrading-and-downgrading-licenses#license-expiration-and-downgrade) for details on the general changes that occur in the event of a license expiration and downgrade.
  * The following plan breakdowns for a detailed comparison between the
  benefits offered with the Free plan and those provided with the trial or paid plans at platform level.
  * Product-specific licensing documentation for a detailed comparison
  between the entitlements offered with the Free plan and those provided with the trial or paid plans. For example, if you want to understand what Orchestrator functionalities you lose access to in the event of a license expiration, refer to the Orchestrator user guide.
  :::

### Application Testing Standard Trial

The Application Testing Standard Trial plan helps you explore how UiPath supports application testing at scale. With this trial you can:

* Use App Tester and App Test Developer licenses to manage most of the testing lifecycle, from test case creation, to execution, and analysis.
* Run tests with App Testing robots to validate application behavior in real environments.
* Leverage Autopilot and agentic capabilities to generate, maintain, and optimize tests.
* Try the extended platform capabilities that come with a paid Application Testing Standard plan.

The evaluation period for the Application Testing Standard Trial is limited to 60 days. After the trial expires, your organization is downgraded to Free.

### Application Testing Standard

The Application Testing Standard plan provides the following benefits:

* Use App Tester and App Test Developer licenses to manage most of the testing lifecycle, from test case creation, to execution, and analysis.
* Run tests with App Testing robots to validate application behavior in real environments.
* Leverage Autopilot and agentic capabilities to generate, maintain, and optimize tests.
* Discover testing opportunities across SAP systems using Change Impact Analysis, which assesses how proposed changes may affect your SAP system and helps you mitigate risks proactively.
* It provides core governance, security, enterprise scale capabilities to enable IT to manage your application testing operations.

### Application Testing Enterprise

The Application Testing Enterprise plan provides the following benefits:

* It provides access to Test Manager Premium Connectors to expand the integration with your ALM (Application Lifecycle Management) tools.
* It provides industry-leading governance, security, and enterprise scale capabilities, such as:
  + Customer Managed Keys: Encrypt your organization data stored across platform using UiPath managed keys or bring your own key to manage the data encryption.
  + External credential store: Offers the ability to configure Orchestrator to access external credential stores. You can store and retrieve sensitive data, such as Robot credentials and credentials assets, when needed.
  + Elastic Robot Orchestration: Allows you to automatically scale your unattended robots by allowing UiPath to scale and manage your robots for you, in the cloud.
  + CI/CD capabilities in Automation Ops: Enable continuous integration and deployment using the built-in Pipelines feature in Automation Ops, or through seamless integration with third-party CI/CD systems for more flexibility and control.

## License types

To access and work with Test Cloud, you need the following license types:

* User licenses: The user licenses which offer access to Test Cloud are:
  + **App Tester - Named User**: Allows you to manage and execute tests, as well as handle other automated processes that enable you to perform application testing.
  + **App Test Developer - Named User**: Allows you to manage and execute tests, as well as handle other automated processes that enable you to perform application testing. Additionally, App Test Developers can create test automations in [Studio STS](https://docs.uipath.com/studio/standalone/latest/user-guide/introduction) or [Studio 2025.10](https://docs.uipath.com/studio/standalone/2025.10) or above, and build Apps.
* Service license: The service license which allow you perform application testing within Test Cloud is the **App Testing Robot** runtime, that allows you to execute automations in Test Cloud.
  :::note
  If you have purchased Test Execution Units, as part of your enterprise Unified Pricing plan, then every job executed by an App Testing Robot is counted as a Test Execution Unit. You can view the number of Test Execution Units in the **Licenses** section of your organization's Administration, under the **Consumables** tab.
  :::

### User licenses

The user licenses that offer access to Test Cloud are:

* **App Tester User - Named User**
* **App Test Developer - Named User**
  :::important
  The licenses associated with Test Cloud (**App Tester User - Named User** and **App Test Developer - Named User**) grant access to all the products available within Test Cloud. No additional user licenses, beyond the **App Tester User** or **App Test Developer**, are needed for accessing the products within Test Cloud.
  :::

The operations that a user can perform in Test Cloud are determined by the type of user license they hold. For specific details regarding the capabilities associated with each license, refer to the **Unified Pricing** licensing sections within the respective product guides.

For general information on user licenses, visit [User licensing](https://docs.uipath.com/overview/other/latest/overview/user-licensing).

### Service licenses

The service license that enables you to work with Test Cloud is the **App Testing Robot** runtime. App Testing Robot is a dedicated runtime license that enables you to run your application test automations within Test Cloud, as well as processes for non-production environments only.

For more information on starting to use App Testing Robots for running your test automations, refer to [Managing machines](https://docs.uipath.com/orchestrator/automation-cloud/latest/user-guide/managing-machines).

:::note
If you have purchased Test Execution Units, as part of your enterprise Unified Pricing plan, then every job executed by an App Testing Robot is counted as a Test Execution Unit. You can view the number of Test Execution Units in the **Licenses** section of your organization's Administration, under the **Consumables** tab.
:::

To learn more about service licenses, refer to [Service licensing](https://docs.uipath.com/overview/other/latest/overview/service-licensing).

## Getting started with Test Cloud licenses

To get started with using Test Cloud, you must meet the following licensing conditions:

* Have a Test Cloud-specific user license assigned: **App Tester User** or **App Test Developer**. These Named User licenses offer you access to the services surfaced through Test Cloud.
* Have **App Testing Robot** runtimes assigned to the tenant where you want to perform application testing.
  :::note
  If you have purchased Test Execution Units as part of your enterprise plan, every job run by your App Testing Robots will count these units. You can view the number of Test Execution Units in the **Licenses** section of your organization's Administration, under the **Consumables** tab.
  :::
