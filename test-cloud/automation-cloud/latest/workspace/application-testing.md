Application testing is available only through Test Cloud. The application testing capabilities of Test Cloud support all stages of the testing process, from the design of test automations to their evaluation and deployment.

1. **Designing test automations**: You can design test automations through various methods only within [Studio Web](https://docs.uipath.com/test-manager/automation-cloud/latest/user-guide/automating-test-cases-in-studio-web), [Studio STS](https://docs.uipath.com/studio/standalone/latest/user-guide/introduction), or [Studio version 2025.10](https://docs.uipath.com/studio/standalone/2025.10) and higher. These include [data-driven testing](https://docs.uipath.com/studio/standalone/latest/user-guide/data-driven-testing), supported by features like Data Service entities, test data files, test data queues, auto-generated data, and AI-generated data. Test automations can be designed using either low-code automations, providing ease of use, or fully [coded automations](https://docs.uipath.com/studio/standalone/latest/user-guide/coded-automations-introduction) for those looking for a more customized experience.
2. **Activities in Studio test automations**: Studio test automations offer robust activities for verifying assertions for your test cases, verifying documents or text equivalence, and generating random data. The activities that the [Testing](https://docs.uipath.com/activities/other/latest/workflow/about-the-testing-activities-pack) and [Mobile Automation](https://docs.uipath.com/activities/other/latest/ui-automation/about-the-mobile-automation-activities-pack) activity packages offer enhance the scope and reliability of your tests.
3. **UI testing**: With the help of [UI Automation](https://docs.uipath.com/activities/other/latest/ui-automation/about-the-ui-automation-activities-pack), you can test UI elements efficiently. The feature leverages an [object repository](https://docs.uipath.com/studio/standalone/latest/user-guide/about-object-repository) to streamline the process and ensure accuracy.
4. **Deployment of test automation projects**: After designing and verifying your test automations, you can easily deploy these as packages to Orchestrator, enabling a smooth transition from test design to test execution.
5. **Test Manager integration**: Test Manager serves as your hub for overseeing all the tests created and deployed through the Studio. It allows you to create requirements, test cases, test sets, and inspect executions. It also supports integration with a range of ALM tools through both [out-of-the-box integrations](https://docs.uipath.com/test-manager/automation-cloud/latest/user-guide/out-of-the-box-connectors) and primarily [Test Manager Connect](https://docs.uipath.com/test-manager/automation-cloud/latest/user-guide/uipath-test-manager-connect). To further boost efficiency, Test Manager uses AI capabilities that [Autopilot<sup>TM</sup>](https://docs.uipath.com/test-manager/automation-cloud/latest/user-guide/about-test-manager) offers for requirement evaluation, manual tests generation from requirements, and review of execution results to generate actionable insights. To get started with Test Manager, check out the available [quickstart guide for Test Manager](https://docs.uipath.com/test-manager/automation-cloud/latest/user-guide/test-manager-quickstart-guide).
6. **Other services in Test Cloud**: In addition to these features, Test Cloud includes other services to support your testing journey. These enhancements make test creation, execution, and evaluation an integrated and streamlined process, reinforcing the quality and performance of your applications. For more information on the services available in Test Cloud, visit [Product offering](https://docs.uipath.com/test-cloud/automation-cloud/latest/workspace/test-cloud-feature-availability#product-availability).
