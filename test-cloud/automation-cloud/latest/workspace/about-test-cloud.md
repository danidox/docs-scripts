UiPath Test Cloud is an enterprise SaaS testing platform designed specifically for application testers. It provides a secure, scalable cloud environment for building, executing, distributing, and managing tests across the organization, without the need to manage infrastructure. Test Cloud is purpose-built for application testing and quality assurance. It brings together test automation, execution orchestration, test management, analytics, and governance into a single, customizable cloud organization. From designing and automating tests to executing them at scale and analyzing results, Test Cloud supports the entire testing lifecycle in one centralized platform.

Test Cloud is available in three flavors, each designed for different operational, compliance, and data residency requirements: the commercially available Test Cloud, Test Cloud Public Sector, and Test Cloud Dedicated.

New products and features are introduced progressively across these cloud offerings. In general, new functionality becomes available first in Test Cloud, and is subsequently made available in Test Cloud Public Sector and Test Cloud Dedicated, subject to compliance, infrastructure, and release readiness requirements. Availability may therefore vary depending on the cloud offering you use. For details, refer to [Feature and product availability](https://docs.uipath.com/test-cloud/automation-cloud/latest/workspace/test-cloud-feature-availability#test-cloud-product-and-feature-availability).

## Test Cloud

Test Cloud is the standard multi-tenant SaaS offering for commercial organizations. It enables teams to start testing within minutes, with minimal setup and no infrastructure management. This flavor is ideal for organizations that want to scale testing quickly, reduce operational overhead, and continuously adopt new testing capabilities.

Key benefits of Test Cloud include:

* **Multi-tenant SaaS efficiency**: Securely share cloud infrastructure while maintaining tenant isolation, ideal for commercial organizations and enterprises.
* **Rapid adoption and expansion**: Designed to help teams grow from initial automation experiments to large-scale deployments quickly.

## Test Cloud Public Sector

Test Cloud Public Sector is an enterprise SaaS testing platform designed for United States public sector organizations. It delivers the full Test Cloud testing experience while meeting stringent government security and compliance requirements.

Public sector organizations often test applications that handle highly sensitive data, such as citizen records and financial information. Test Cloud Public Sector is built to support these needs and holds **FedRAMP Moderate authorization**, demonstrating compliance with federal security standards.

Key benefits of Test Cloud Public Sector include:

* **FedRAMP Moderate compliance**: Meets U.S. federal security and compliance requirements for handling sensitive government data.
* **Strong encryption standards**: Uses FIPS 140-2 validated encryption for data at rest and in transit, with TLS 1.2 or higher for secure communications.
* **Controlled access model**: UiPath backend access is restricted to U.S. citizens and permanent residents operating within the United States.
* **Secure platform components**: Services run on hardened images to reduce vulnerabilities and strengthen overall security posture.

## Test Cloud Dedicated

Test Cloud Dedicated is a single-tenant enterprise SaaS testing platform for large organizations with advanced security, compliance, and data residency requirements. It provides a dedicated testing environment with isolated infrastructure, while maintaining the simplicity and scalability of a managed cloud platform.

This flavor is designed for organizations that require greater operational control or test applications involving highly sensitive data, enabling them to scale testing operations while meeting internal and regulatory obligations.

Key benefits of Test Cloud Dedicated include:

* **Single-tenant isolation**: Dedicated compute and data resources provide infrastructure-level isolation and predictable performance.
* **Upgrade control and stability**: Control when updates are applied using delayed releases and blackout windows of up to 30 days.
* **Data residency choice**: Deploy your dedicated instance in an Azure region of your choice to meet data locality and regulatory requirements.
* **Custom access and connectivity**: Use a custom fully qualified domain name and configure secure connectivity options, including restricted CIDR ranges and private access models.
