Use Autopilot<sup>TM</sup> to import manual test cases from Excel files. The import process transfers all information to Test Manager, unless specified otherwise. For instance, test case properties such as Priority, Status, or Owner are imported into Test Manager as custom field values at the test case level.

To import manual test cases, navigate to the **Test Cases** section of your project, and select **Import test cases** > **Start new import**. Upload your document with the manual tests and provide any additional instructions. Preview the imported tests in order to review them. After reviewing, refine the instructions, if necessary, and re-import the test cases. When you are satisfied with the results, select the desired tests and create them in Test Manager.

To better identify the imported test cases, you can instruct Autopilot, in the **Provide import instructions** section, to place certain labels on the imported test cases.

To check how to import manual test cases, visit [Importing manual test cases](https://docs.uipath.com/test-suite/automation-cloud/latest/user-guide/importing-manual-test-cases).
