:::note
This capability is available only in Test Manager delivered via Test Cloud.
:::

Use Autopilot<sup>TM</sup> to identify test cases that may no longer be relevant to an updated requirement. Autopilot analyzes test case coverage, looking for deprecated functionality, redundant steps, unsupported environments, or outdated logic.

Open a requirement and select **Optimize coverage** > **Find obsolete tests** to initiate the review.

You can enhance the analysis by adding supporting documents and custom instructions to guide Autopilot. Once the review completes, you are notified of any test cases flagged as obsolete, which you can then unassign or label as needed.

Visit [Finding obsolete tests based on requirements](https://docs.uipath.com/test-manager/automation-cloud/latest/user-guide/finding-obsolete-tests-based-on-requirements) for detailed steps.
