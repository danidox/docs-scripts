Studio Web projects of type **Tests** offer the same management and collaboration experience as RPA workflow projects. Once your Tests project is created, either by you or shared with you, it becomes accessible on the **Workspace** page. From the Workspace you can manage, share, collaborate on, and execute tests.

For more detailed information about managing project files, project sharing, collaboration, templates, and automation details, refer to [Managing RPA projects](https://docs.uipath.com/studio-web/automation-suite/latest/everyone-user-guide/managing-projects).

Key actions that you can perform are:

* Navigate to your Workspace to view recently accessed Tests projects. Use filters, sorting, and pinning to organize your project list.
* Open a project directly in Studio Web. Share the project with teammates for collaboration and troubleshooting.
  :::note
  Only one user can edit a shared project at a time.
  :::
* Run or debug tests directly in the web interface. You can view test results and logs in real-time.
* Publish the Tests project to Orchestrator to make the test cases available for execution via pipelines.
* Create backups or clones of your Tests projects by downloading or duplicating them.
* Convert a Tests project into a reusable template for your organization, and manage these templates from the Studio Web Templates page.
