The following schema describes a simplified process of designing and consuming API workflows:

1. **Build workflows in Studio Web**
   * Design API workflows using the API-centric designer canvas in Studio Web.
2. **Publish workflows to Orchestrator**
   * API workflows are published as Orchestrator processes that run as a serverless function.
3. **Consume workflows across platforms**
   * Integrate and reuse workflows throughout the UiPath Platform.
   * Leverage time, event, and API triggers to run API workflows processes from Orchestrator.
